# Local Development

When developing Gomoku locally, `Docker` is essential.

## 1. Clone github repo

```bash
> git clone git@github.com:sungyongcho/gomoku.git
> cd gomoku
```

## 2. Apply alias.sh

```bash
> source alias.sh
```

## 3. Create `.env` in project root

```.env
LOCAL_FRONT=3000
LOCAL_FRONT_NUXT_CONTENT_WS=4000
LOCAL_MINIMAX=8005
LOCAL_MINIMAX_GDB=8006
LOCAL_ALPHAZERO=8080

FRONT_WHERE=local
DEPLOY_ALPHAZERO_CONFIG=configs/deploy.yaml # edit num_search, num_simulations, etc. for deployment
DEPLOY_ALPHAZERO_DEVICE=cpu
```

Local AlphaZero serving uses `alphazero/models/champion.pt` as the latest champion model checkpoint.
This documentation is published at `sungyongcho.com/gomoku`, and the source code is in `github.com/sungyongcho/gomoku`.
`alphazero/models/champion.pt` is versioned in that GitHub repository, so cloning the repository includes the checkpoint by default.

## 4. Launch Application

```bash
> dev-up
```
